# 실시간 화재 탐지 시스템 (Real-Time Fire Detection System)

## 📋 프로젝트 개요

CCTV 영상에서 화재와 연기를 실시간으로 탐지하고, LLM을 통해 구역 관리자 및 건물 관리자에게 신고 방법과 대처 방법을 안내하는 시스템입니다.

| 항목 | 상세 정보 |
|------|---------|
| **기술 스택** | YOLOv11m, Object Detection, Anaconda |
| **프로젝트 규모** | 개인 프로젝트 |
| **프로젝트 기간** | 2025/09/01 ~ 2025/12/20 |
| **프로젝트 역할** | 딥러닝 모델 개발 및 성능 개선 |
| **주요 목표** | 화재 및 연기 객체 실시간 탐지 모델 개발 |
| **GitHub** | [Fire-Detection](https://github.com/rxu0301/Fire-Detecion) |

---

## 1️⃣ 연구 배경 및 목적

화재는 초기 대응이 무엇보다 중요한 재난 요소입니다. 본 프로젝트는 다음을 목적으로 수행되었습니다:

- 실시간 환경에서 적용 가능한 **경량 화재 탐지 모델** 설계
- **데이터 증강 및 정규화 전략**이 탐지 성능에 미치는 영향 비교·분석
- 탐지 속도와 정확도를 동시에 만족하는 모델 구현

### 모델 선정 이유

**YOLOv11n을 선택한 이유:**
- 단일 네트워크 기반의 빠른 객체 탐지 구조
- 속도와 정확도 간 우수한 균형
- CCTV, 드론 등 엣지 디바이스 환경에 최적화된 경량 모델
- 연산량 최소화로 실시간 처리 가능

---

## 2️⃣ 모델 구조 및 알고리즘

### YOLOv11 아키텍처

YOLOv11은 다음 세 가지 모듈로 구성됩니다:

#### **Backbone**
- 입력 영상으로부터 다계층 특징 추출
- 작은 불꽃, 연기 등 경계가 불명확한 패턴 포착

#### **Neck**
- FPN(Feature Pyramid Network)과 PAN(Path Aggregation Network) 활용
- 서로 다른 스케일의 특징 통합
- 작은 화염부터 대규모 화재까지 동시 탐지 가능

#### **Head**
- Bounding Box 회귀 및 클래스 확률 동시 예측
- DFL(Distribution Focal Loss) 기반 위치 보정으로 정확도 향상

**주의사항:** 구조 변경 없이 기본 아키텍처를 유지하고, 학습 전략 차이에 따른 성능 변화를 비교 분석했습니다.

## 4️⃣ 실험 설계

### 학습 전략 비교

총 3가지 학습 전략을 설계하여 실험을 진행했습니다:

| 실험 | 모델 구조 | Epochs | Optimizer | Learning Rate | 데이터 증강 |
|------|---------|--------|-----------|--------------|-----------|
| **Baseline** | YOLOv11n | 100 | SGD | 0.01 | ✗ |
| **Augmentation** | YOLOv11n | 100 | SGD | 0.01 | ✓ |
| **Regularization** | YOLOv11n | 100 | AdamW | 0.01 | ✓ |

### 4.1 Baseline 학습

```
- 초기 가중치: yolo11n.pt (사전학습 모델)
- Epochs: 100
- Optimizer: SGD (momentum=0.937)
- Batch Size: 16
- Learning Rate: 0.01
- 목적: 모델의 기본 성능 확인 및 기준 설정
```

### 4.2 데이터 증강 학습

```
- 기반 모델: Baseline의 best.pt
- 추가 학습: 20 Epoch
- 증강 기법:
  - Mosaic
  - Mixup (α=0.2)
  - AutoAugment
  - Erasing
  - FlipLR
- 목적: 다양한 화재 환경에 대한 일반화 성능 확보
```

### 4.3 정규화 학습

```
- Optimizer: AdamW
- Weight Decay: 0.0005
- Warm-up Epoch: 3
- 데이터 증강: Augmentation과 동일
- 목적: 과적합 방지 및 수렴 안정성 확보
```

## 5️⃣ 성능 비교 및 분석

### 실험 결과

| 실험 | Precision | Recall | mAP@0.5 | mAP@0.5-0.95 |
|------|-----------|--------|---------|--------------|
| **Baseline** | 0.9353 | 0.8495 | **0.9258** | 0.8385 |
| **Augmentation** | 0.9431 | **0.9237** | 0.9191 | 0.8255 |
| **Regularization** | **0.9455** | **0.9249** | 0.9257 | **0.8300** |

### 주요 결과 해석

#### **Baseline의 한계**
- 높은 mAP@0.5 달성
- **낮은 Recall (0.8495)**: 일부 화재 객체를 놓치는 경향
- 실제 재난 탐지에서 중요한 한계점

#### **데이터 증강의 효과**
- **Recall 대폭 향상** (0.8495 → 0.9237)
- 다양한 시각적 변형이 모델의 일반화 능력 개선
- 다양한 환경 조건 적응성 증대

#### **정규화 학습의 효과**
- **손실 곡선 안정화** (진동 최소화)
- **Precision 및 Recall 균형 개선**
- AdamW + Weight Decay = 안정적인 가중치 업데이트
- 과적합 가능성 감소
- 수렴 안정성 향상

### 최종 평가

**Regularization 전략**이 가장 우수한 결과를 도출했습니다:
- Precision과 Recall의 균형 (0.9455, 0.9249)
- 안정적인 학습 곡선
- 실시간 환경에서의 신뢰성 확보

## ❓ Q&A

### Q1. 왜 YOLOv11n을 선택했는가?

실시간 탐지가 목적이었기 때문에 경량 구조가 필수적이었습니다. YOLOv11n은 **연산 효율성과 정확도 간 균형이 우수**하여 엣지 환경 적용 가능성이 높았습니다.

### Q2. Baseline에서 가장 아쉬웠던 점은 무엇인가?

mAP는 높았지만 **Recall이 낮았습니다** (0.8495). 이는 일부 화재를 놓칠 가능성을 의미하며, 실제 재난 탐지에서는 중요한 한계점입니다.

### Q3. 데이터 증강은 어떤 변화를 만들었는가?

**Recall이 크게 개선되었습니다** (0.8495 → 0.9237). 다양한 환경 조건을 가정한 학습이 모델의 탐지 범위를 확장시켰다고 판단됩니다.

### Q4. 정규화 학습의 핵심 효과는 무엇이었는가?

**손실 곡선의 안정화**였습니다. AdamW와 Weight Decay는 가중치 업데이트를 안정화시켜 과적합 가능성을 줄였습니다.
